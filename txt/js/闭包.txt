（闭包原理，底层解析，存在的问题）

闭包原理：
	闭包就是在函数中定义的函数，是连接函数内外的桥梁。
	
	这里就应用到作用域链的知识，当前作用域可以访问外部作用域的参数、变量和函数，故子函数可以访问父函数里的数据
	
	变量有全局变量和局部变量，全局变量任何地方都能访问，局部变量只有在特点作用域中才能访问到。（注意，局部变量声明必须var，吧var的话相当于声明一个全局变量）


存在问题：
	内存泄露(父函数中数据不被垃圾回收机制回收）
	
	f1是f2的父函数，f2被定义为全局变量，导致f2始终在内存中，f2依赖于f1，所以f1也是在在内存中，不会在调用结束后辈垃圾回收机制回收。

	所以不可以滥用闭包，有安全隐患，还有就是内存过多的占用，会造成浏览器性能问题。



注意要点：
	1. 闭包会把函数中变量保存在内存中，内存消耗过大，会内存泄露和性能差的问题，所以，在退出函数之前，将不使用的变量删除
	2. 闭包会使在外部可以改变父函数内部的数据，所以要注意。不能随便改值。
function handler(){
    var ele = document.getElementById("ele");
    var id = ele.id;
    ele.onclick = function(){
        alert(id);
    }
    ele = null;
}
